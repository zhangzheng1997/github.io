    标识符

    程序中，自己定义的内容。类的名字、方法的名字和变量的名字

    1、包含英文字母（区分大小写）、数字、$、_

    2、不能以数字开头

    3、不能是关键字

    常量

    字符串： 双引号引起的部分，叫做字符串常量 “abc”

    整型：直接的数字，没有小数 100、200

    浮点数

    字符常量：单引号引起的单个字符 ‘a’、‘9’

    布尔：true、false

    空常量：null

    基本数据类型

    整数 byte short int long

    浮点 float（四字节） double（八字节）

    字符 char

    布尔型 boolean

    引用数据类型

    字符串、数组、类、接口、Lambda

    注意事项：

    1、字符串不是基本类型，而是引用类型

    2、浮点数可能只是一个近似值，并非精确的值

    3、数据范围与字节数不一定相关，float是四字节，long是八字节

    4、浮点数默认是double，要用float，要加上一个后缀F

    整数默认int，要使用long，加入后缀L

    方法入门

    方法定义

    public static void 方法名称 () {

    ​ 方法体

    }

    方法名称的命名规则和变量一样，使用小驼峰

    方法体：也就是大括号中可以包含任意语句。

    IDEA

    Create project

    选择empty project

    modules + new modules

    psvm 回车 一个m 回车就可以

    ctrl+alt+L 格式化代码

    导入模块

    file ->project structure -> + -> import modules

    比如5.fori就是for循环

    定义方法的完整格式：

    修饰符 返回值类型 方法名称（参数类型 参数名称，）{

    ​ 方法体

    ​ return 返回值

    }

    修饰符：现阶段的固定写法，public static

    方法的三种调用：

    1、单独调用：方法名称（参数）

    2、打印调用

    3、赋值调用

    方法重载（overload）

    多个方法的名称一样，但是参数列表不一样。

    数组的特点：
    1、数组是一组引用数据类型

    2、数组当中的多个数据，类型必须统一

    3、数组的长度在程序运行期间不可改变

    两者初始化方式：

    1、动态初始化（指定长度）

    2、静态初始化（指定内容）

    动态初始化数组的格式：

    数据类型[] 数组名称 =new 数据类型[数组长度];

    int[] array;

    array=new int[5];

    静态初始化基本格式：

    数据类型[] 数组名称 =new 数据类型[] {元素1，元素2，、、};

    int[] array;

    array=new int[] {10,11,12};

    省略格式：int [] array={10,11,12};

    动态初始化数组的时候，其中元素会拥有一个默认值。

    整数，默认为0；

    浮点数，默认为0.0；

    字符，默认为‘\u0000’；

    布尔，默认为false；

    饮用，默认为null。

    java内存需要划分为5个部分：

    1、栈（stack）：存放的都是方法中的局部变量。

    方法的运行一定要在栈中

    ​ main

    局部变量：方法中的参数或是方法{}内部的变量

    ​ 作用域：一旦超出作用域，立刻从内存中消失

    2、堆（heap）：凡是new出来的东西，都在堆当中

    ​ 堆内存里面的东西都有一个地址值：16进制

    3、方法去：存储class相关信息，包含方法的信息。

    4、本地方法栈：与操作系统相关

    5、寄存器：与cpu相关

    数组作为方法的参数。

    当调用方法的时候，像方法的小括号进行传参，传递进去的其实是数组的地址值。

    一个方法有0、1、多个参数，但是只能有0或者1个返回值，不能有多个返回值。

    如果希望一个方法中产生多个结果数据进行返回，使用一个数组作为返回值类型。

    类与对象

    类

    一组相关属性和行为的集合

    属性：事物的状态信息

    行为：事物能够做什么

    类是对象的模板，对象是类的实体

    成员变量：对应事物的属性

    成员方法：对应事物的行为

    一个类并不能直接使用，需要根据类创建一个对象，才能使用。

    1、导包：也就是指出需要使用的类，在什么位置

    Import 包名类、类名称

    2、创建，格式

    类名称 对象名=new 类名称（）

    3、使用，分为两种：

    使用成员变量：对象名.成员变量名

    使用成员方法：对象名.成员方法名（参数）

    局部变量与成员变量

    1、定义位置不一样

    局部变量：在方法内部

    成员变量：在方法的外部，直接写在类中

    2、作用范围不一样

    局部变量：只有方法中可以使用，出了方法就不能使用

    成员变量：整个类可以用

    3、默认值不一样

    局部变量：没有默认值，如果想使用，必须手动进行赋值

    成员变量：如果没有赋值，会有默认值，规则和数组一样

    4、内存位置不一样

    局部变量：位于栈内存

    成员变量：堆内存

    5、生命周期

    局部变量：随着方法进栈而诞生，随着方法出栈而消失

    成员变量：随着对象创建而诞生，随着对象垃圾回收而消失

    面向对象三大特征：封装、继承、多态

    封装性在Java中的体现：

    1、方法就是一种封装

    2、关键字private也是一种封装

    封装就是将一些细节信息隐藏起来，对外界不可见。

    当方法的局部变量和类的成员变量重名的时候，依据“就近原则”，优先使用局部变量。如果需要访问本类当中的成员变量，需要使用格式：this.成员变量名

    构造方法是专门用来创建对象的方法，但我们通过关键字new来创建对象时，其实就是在调用构造方法。

    注意：
    1、构造方法的名称必须和所在的类名称完全一样，就连大小写也一样

    2、构造方法不需要写返回值类型，连void都不写

    3、构造方法不能return一个具体的返回值

    4、如果没有编写任何构造方法，那么编译器将会默认一个构造方法，没有参数

    一个标准的类通常拥有下面四个部分

    1、所有的成员变量都要使用private关键字修饰

    2、为每个成员变量编写一对Getter/Setter方法

    3、编写一个无参数构造方法

    4、编写一个全参数构造方法

    这样的标准类叫做java bean
